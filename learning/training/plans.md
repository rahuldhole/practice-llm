<summary><strong>Appendix: Output Artifacts Explained</strong></summary>

* **Foundational Model**: A large-scale base model trained from scratch or raw inputs, possessing general knowledge and reasoning capabilities.
* **Tokenizer Vocab**: A trained vocabulary mapping that fragments raw text or specialized syntax into discrete tokens for a model to process.
* **Merged Base Model**: A new base model created by interpolating the weights of two or more existing models, combining their capabilities without additional compute.
* **Upgraded Base Model**: An existing base model whose architecture has been structurally modified (e.g., dense to MoE or attention changes) and stabilized.
* **Distilled Model**: A lightweight model trained to mimic the reasoning patterns or probability distributions of a much larger, more capable teacher model.
* **Domain-Adapted Model**: A base model that has undergone continual pre-training on a massive corpus of domain-specific text to deeply internalize new knowledge.
* **Expanded Base Model**: A base model that has been artificially enlarged (e.g., through layer stacking) and then trained to stabilize the new architecture.
* **Aligned Model**: A base model fine-tuned using human or AI preferences (e.g., RLHF, DPO) to follow rules, adopt a specific tone, or ensure safe behavior.
* **Fine-Tuned Model**: A model that has been supervised on specific prompt-response pairs to learn specialized tasks (e.g., formatting, tool use, or reasoning traces).
* **Reward Model**: A specialized model trained to score or evaluate the quality of responses, typically used as an intermediate step for reinforcement learning.
* **Adapter Weights**: Extremely lightweight, trainable parameter matrices (e.g., LoRA) overlaid on a frozen base model to adapt it without modifying original weights.
* **Bottleneck Adapters**: Small neural network layers inserted between the frozen layers of a base model to enable modular, plug-and-play capabilities.
* **Synthetic Dataset**: A large collection of high-quality training data generated algorithmically by frontier models rather than collected from human annotators.
* **Multi-Modal Adapter**: Specialized projection layers trained to bridge different modalities (like images or audio) into the text embedding space of a base model.
* **Data Mixture Weights**: A learned set of proportional weights that dictate how much data from different domains (e.g., code, web) should be included in a training run.
* **Pruned Model / Pruned Base Model**: A model that has had less-important weights or channels permanently removed to reduce disk footprint and increase inference speed.
* **Private Base Model**: A base model trained using mathematical privacy guarantees (DP-SGD) or federated methods to prevent the memorization of sensitive data.
* **Curated Dataset**: A high-quality pool of training data systematically selected from a larger pool, often using active learning to find the most valuable samples.
* **Quantized Model**: A model whose weights have been heavily compressed (e.g., to 2-bit or 4-bit) during training to maintain capability at extreme edge sizes.
* **MTP Base Model**: A base model trained with secondary prediction heads to output multiple tokens simultaneously, massively boosting decoding speeds.
* **Drafter Model**: A hyper-lightweight model trained specifically to mimic the token distribution of a larger model to accelerate generation via speculative decoding.
* **Compiled Kernel**: Low-level, hardware-specific graph executable instructions optimized to run a model efficiently on local NPUs (Neural Processing Units).
* **Locally-Routed MoE**: An edge-optimized MoE model whose routing mechanisms have been adapted to run within tight memory constraints to avoid latency.

</details>
